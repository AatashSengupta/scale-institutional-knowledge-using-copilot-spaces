# Cross-Functional Role Integration Checklist

## Purpose
Ensure all key roles are integrated in project planning, delivery, and release activities. This checklist helps teams maintain comprehensive stakeholder alignment and clear accountability across the project lifecycle.

## Project Initiation Phase

### Role Involvement
- [ ] **Product Manager**: Confirm problem statement and define customer transformation objectives
- [ ] **Project Manager**: Identify cross-team coordination tasks and dependency patterns
- [ ] **UX Designer**: Conduct initial user research and identify potential usability pain points
- [ ] **Data Analyst**: Define initial success metrics and tracking methodology
- [ ] **DevOps Engineer**: Assess infrastructure needs and CI/CD requirements
- [ ] **Technical Writer**: Identify documentation needs and user guide scope
- [ ] **QA Lead**: Provide initial quality assurance strategy considerations
- [ ] **All stakeholders**: Aligned on project charter, goals, and success criteria

## Planning Phase

### Scope & Requirements
- [ ] **Developers & Architects**: Define technical architecture and estimate effort
- [ ] **UX Designer**: Create wireframes and user flow acceptance criteria
- [ ] **QA Lead**: Develop comprehensive test strategy and acceptance criteria matrix
- [ ] **Data Analyst**: Define specific tracking points and metric collection methods
- [ ] **DevOps Engineer**: Define deployment environments and CI/CD pipeline stages
- [ ] **Technical Writer**: Outline documentation structure and content scope
- [ ] **Project Manager**: Coordinate dependencies, timelines, and resource allocation
- [ ] **Product Manager**: Map desired outcomes to acceptance criteria for each phase

### Definition of Done (DoD)
- [ ] **Cross-functional team**: Jointly define DoD including:
  - Code review and test coverage requirements (Developers, QA Lead)
  - Design approval and accessibility compliance (UX Designer)
  - Metric tracking implementation (Data Analyst)
  - Documentation completion (Technical Writer)
  - Deployment readiness checklist (DevOps Engineer)
  - Stakeholder communication and approvals (Project Manager)

## Execution Phase

### Daily/Weekly Standups
- [ ] **Developers**: Report progress, blockers, code review status
- [ ] **UX Designer**: Conduct feature usability testing and document feedback
- [ ] **QA Lead**: Execute test cases and triage defects
- [ ] **Data Analyst**: Track project velocity and sprint progress metrics
- [ ] **DevOps Engineer**: Monitor CI/CD pipeline health and deployment readiness
- [ ] **Technical Writer**: Address feature documentation in parallel with development
- [ ] **Project Manager**: Track risks, dependencies, and escalations
- [ ] **Product Manager**: Evaluate features against success criteria and pivot if needed

### Quality Assurance & Testing
- [ ] **QA Lead**: Manage test execution against acceptance criteria
- [ ] **UX Designer**: Conduct usability testing and gather user feedback
- [ ] **Developers**: Address defects and implement fixes
- [ ] **Data Analyst**: Validate metric tracking is working correctly
- [ ] **DevOps Engineer**: Ensure test environments mirror production
- [ ] **Technical Writer**: Update documentation based on implementation changes

### Risk & Dependency Management
- [ ] **Project Manager**: Maintain risk register and escalation paths
- [ ] **Technical leads** (Developers, DevOps, QA): Identify technical risks
- [ ] **Product Manager**: Highlight product/market risks
- [ ] **All roles**: Flag cross-functional dependencies in sprint planning
- [ ] **Project Manager**: Review escalation status at weekly syncs

## Release Phase

### Pre-Release Validation
- [ ] **QA Lead**: Confirm complete test coverage and bug resolution
- [ ] **Technical Writer**: Complete all release documentation and release notes
- [ ] **Data Analyst**: Confirm metrics are tracking correctly in production environment
- [ ] **DevOps Engineer**: Validate CI/CD pipeline readiness and rollback plan
- [ ] **Developers**: Perform final smoke tests and document troubleshooting steps
- [ ] **UX Designer**: Validate final user experience in staging/pre-production
- [ ] **Project Manager**: Confirm stakeholder communication and launch coordination
- [ ] **Product Manager**: Approve launch announcement and success criteria targets

### Deployment & Monitoring
- [ ] **DevOps Engineer**: Execute deployment and run post-deploy verifications
- [ ] **QA Lead**: Run production smoke tests and monitor critical user paths
- [ ] **Data Analyst**: Monitor metric collection and establish baseline performance
- [ ] **Developers**: Stand by for immediate troubleshooting if needed
- [ ] **Project Manager**: Coordinate communications to support and stakeholders
- [ ] **Product Manager**: Monitor customer adoption and early feedback
- [ ] **Technical Writer**: Publish release documentation and user announcements

### Post-Release Monitoring
- [ ] **DevOps Engineer**: Monitor system performance and error rates
- [ ] **Data Analyst**: Track key metrics against success criteria targets
- [ ] **Developers**: Triage and fix production issues
- [ ] **QA Lead**: Validate production quality and user experience
- [ ] **Project Manager**: Collect initial feedback and status for stakeholders
- [ ] **Product Manager**: Assess market reception and customer satisfaction

## Retrospective Phase

### Post-Release/Project Review
- [ ] **All roles**: Participate in retrospective to capture learnings
- [ ] **Data Analyst**: Present measured impact and key metrics findings
- [ ] **Project Manager**: Summarize schedule, scope, and resource performance
- [ ] **Product Manager**: Share customer feedback and product adoption data
- [ ] **Developers**: Highlight technical learnings and architectural insights
- [ ] **QA Lead**: Discuss quality trends and testing effectiveness
- [ ] **UX Designer**: Present usability findings and user satisfaction data
- [ ] **DevOps Engineer**: Discuss deployment reliability and infrastructure learnings
- [ ] **Technical Writer**: Gather feedback on documentation effectiveness
- [ ] **All roles**: Identify action items and assign owners for improvements

### Best Practices Preservation
- [ ] **Project Manager**: Document project runbook and decision history
- [ ] **Technical Writer**: Capture and archive institutional knowledge
- [ ] **Data Analyst**: Preserve key metrics, baseline data, and benchmarks
- [ ] **Developers**: Update architectural and technical documentation
- [ ] **All roles**: Update role-specific playbooks and checklists based on learnings

---

## Notes
- Use this checklist to ensure no role is overlooked during critical project phases
- Customize based on your team size and project scope (not all roles may be present in every project)
- Review and update this checklist after each project to incorporate new learnings
- Reference the `octoacme-roles-and-personas.md` document for detailed role responsibilities and communication patterns
